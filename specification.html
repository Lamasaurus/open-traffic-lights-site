Publishing real-time data

The status plan of a crossing can change every X subseconds. To enable clients to retrieve these updates as fast as possible while keeping the server cost low, a publishing mechanism is proposed by relying on standards HTTP/2 and SSE.

Server-Sent Events (SSE)

SSE is a very simple method for publishers to push events over HTTP to clients:

  • A client sends a request to a server through the EventSource interface that contains the SSE API.
  • The server keeps the connection open, sets the mime-type to text/event-stream and streams events back.
  • That's it. Both the server as the client can choose to close the stream.

HTTP/2

This new version of HTTP has the advantage to combine streams over the same connection, which is called multiplexing. This means that a client can dynamically subscribe to multiple streams (read: traffic lights) without opening/closing a TCP connection every time.

This also takes away the disadvantage of SSE over HTTP/1.1 where the browser has a limited amount of connections to the same domain.
